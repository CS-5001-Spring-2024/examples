Student

User Interface

You will implement a grade_book.py program that will, at minimum, implement the functionality from Lab 7 as described below. 

  1. The user may record data for as many students as they would like.  After each student, the program will ask "Would you like to record scores for another student? (y/n)". As long as the user says 'y' the program will continue.
  2. The user may record as many scores as they wish for each student. When they are finished entering scores for a student they will enter -1 to end input.
  3. The program must handle invalid user input at any stage. This includes (1) score is not a floating point number; (2) user does not enter 'y/n' when asked if they want to enter scores for another student.
  4. Invalid input will be handled by asking the user to enter input again.
